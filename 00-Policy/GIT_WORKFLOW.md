# Git Workflow

- Use Feature Branch Workflow: create a new branch from `main` or `develop` for every feature or bug fix.
- Branch naming convention: `feature/<module-name>`, `bugfix/<issue-number>`, `hotfix/<description>`.
- Commit messages should be clear and concise.
- All branches must be merged via Pull Requests. Code review is mandatory before merging.
- Do not commit directly to `main` or `develop`.
